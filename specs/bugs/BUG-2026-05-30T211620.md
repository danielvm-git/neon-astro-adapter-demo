# BUG-2026-05-30T211620: SSR function crashes when Neon Auth env vars are unset

## Problem

Deploying to Netlify without setting `NEON_AUTH_BASE_URL` and `NEON_AUTH_COOKIE_SECRET` causes all routes to return HTTP 500. The user sees a browser download prompt or an empty error page.

The build succeeds locally (env vars are present in `.env`), but the deployed SSR function crashes on every request because the middleware calls `createAstroAuth()` at module scope with undefined config values, and `validateConfig` inside the auth package throws immediately.

**Actual behavior:** HTTP 500 on all routes (`/`, `/api/health`).

**Expected behavior:** Site serves content with graceful degradation — health checks report auth as "missing" and the app works without authentication configured.

**Reproduction:** Deploy to Netlify without setting `NEON_AUTH_BASE_URL` or `NEON_AUTH_COOKIE_SECRET` env vars. Visit any URL.

## Root Cause Analysis

The middleware (`src/middleware.ts`) initializes `createAstroAuth` at module load time:

```ts
const authMiddleware = createAstroAuth({
  baseUrl: import.meta.env.NEON_AUTH_BASE_URL,
  cookies: { secret: import.meta.env.NEON_AUTH_COOKIE_SECRET },
  skipRoutes: [...],
}).middleware();
```

When `NEON_AUTH_BASE_URL` is undefined, `@danielvm/neon-astro-auth`'s `validateConfig` throws `"Missing required config: baseUrl."`. This error is uncaught during module initialization when the SSR function starts handling requests.

The call chain is:
1. Netlify SSR function receives a request
2. Astro bootstraps middleware
3. Module-level code runs `createAstroAuth()` → `validateConfig()` → **throws**
4. Function handler catches nothing → HTTP 500

The `skipRoutes` list (`"/"`, `"/login"`, `"/signup"`, `"/api/health"`) is irrelevant because the crash happens before any routing logic runs.

**Contributing factors:**
- `import.meta.env` in SSR functions reads from the runtime environment, not `.env` files
- Netlify's Framework API SSR function does not inject `.env` during local builds
- Local development works because `.env` exists on disk
- Risk level: **High** (production site completely down without env vars)

## Resolution

**Fixed:** 2026-05-30
**Root cause confirmed:** `createAstroAuth()` called at module scope with `undefined` env vars; `validateConfig` throws immediately, crashing the SSR function on every request.
**Fix applied:** Wrapped initialization in `getAuthMiddleware()` guard that checks `NEON_AUTH_BASE_URL` and `NEON_AUTH_COOKIE_SECRET` before calling `createAstroAuth`. When either is missing, a no-op middleware is returned.
**Hardening added:** Environment check at startup (`getAuthMiddleware` guard)
**Evidence:** all tests pass (`npx vitest run`)
**Commit:** `fix(middleware): guard against missing Neon Auth env vars`
